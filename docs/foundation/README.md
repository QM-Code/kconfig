# Foundation Docs

`docs/foundation/` is the canonical home for long-lived documentation.

Use this tree for material that should outlive transient execution tracks:
- policy (`docs/foundation/policy/`): invariants, execution rules, durable decisions,
- governance (`docs/foundation/governance/`): overseer workflow, handoff process, CI/test governance,
- architecture (`docs/foundation/architecture/`): rewrite-owned technical contracts and models.

Rules:
- Do not keep transient project status/assignment state here.
- Do not duplicate policy/governance/architecture rules in `docs/projects/`.
- Treat `docs/projects/` as transient execution plans that can be archived without losing durable knowledge.
